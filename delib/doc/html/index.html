Introduction

Cette bibliothèque présente une implémentation de l'algorithme de l'évolution différentielle, une méthode d'optimisation stochastique et globale. L'algorithme vise à optimiser un problème en améliorant de manière itérative une solution candidate avec le temps. Il se base sur les concepts de mutation, croisement et sélection, inspirés par le processus biologique de l'évolution naturelle.

L'implémentation proposée repose fortement sur le paradigme de programmation orientée objet.

Algorithme d'évolution différentielle

Ce projet considère que l'étudiant possède déjà des connaissances de base sur les concepts et l'exploitation de l'algorithme d'évolution différentielle.

Le résumé suivant rappel brièvement les notions importantes. L'algorithme d'évolution différentielle est particulièrement efficace pour les problèmes d'optimisation complexes. Il travaille avec une population de solutions candidates qui évoluent au fil des générations pour trouver une solution optimale ou satisfaisante. À chaque génération, de nouvelles solutions sont créées par combinaison et mutation des solutions existantes, puis sélectionnées pour la prochaine génération selon leur performance ou leur fitness.

La différence majeure entre cette implémentation et celle qui a été réalisée au laboratoire 2 est l'utilisation de toutes les notions du paradigme orienté objet (encapsulation, héritage et polymorphisme) avec une emphase particulière sur le polymorpshisme dynamique.

Concept de stratégie

Dans le cadre de ce projet, le patron de conception stratégie est utilisé pour définir plusieurs familles d'algorithmes, encapsuler chacun d'eux comme un objet, et les rendre interchangeables.

Cette approche flexible permet aux algorithmes de varier indépendamment des clients qui les utilisent. Cette approche est avantageuse pour l'algorithme d'évolution différentielle car elle permet une flexibilité et une réutilisabilité accrues des différentes stratégies (initialisation, mutation, croisement, sélection, etc.) selon les besoins spécifiques du problème à résoudre.

Constituants principaux de la bibliothèque

La librairie utilise le namespace de.

Voici les classes importantes à connaître :

  • de::DifferentialEvolution, le moteur de l'évolution différentielle (la classe centrale)
  • de::EngineParameters, la classe de configuration des paramètres spécifiques au moteur de l'évolution différentielle
  • de::SolutionStrategy, la classe représentant le problème à résoudre *

Il existe de nombreux algorithmes qui sont généralisés via la notion de stratégie. Voici les classes importantes définissant ces algorithmes et leurs implémentations disponibles:

  • de::InitializationStrategy, la stratégie d'initialisation du domaine au tout début de l'évolution *
    • de::InitializationDomainUniform, initialise aléatoirement le domaine uniformément, la stratégie par défaut
    • de::InitializationBySolution, initialise par le comportement défini à même la solution
  • de::AdaptiveDomainStrategy, la stratégie de gestion du domaine pendant l'évolution *
    • de::AdaptiveDomainNoConstraint, aucune contrainte de domaine pendant l'évolution
    • de::AdaptiveDomainClamped, contraint la solution au domaine pendant l'évolution (limite aux bornes du domaine)
  • de::FitnessStrategy, la stratégie du calcul de la fitness *
    • de::FitnessIdentity, la fitness identité, la stratégie par défaut
    • de::FitnessInverse, la fitness inverse
    • de::FitnessOpposite, la fitness opposée
    • de::FitnessLinear, la fitness linéaire
  • de::OptimizationStrategy, la stratégie d'optimisation (minimisation ou maximisation) *
    • de::OptimizationMaximization, retient la solution ayant une valeur plus élevée, la stratégie par défaut
    • de::OptimizationMinimization, retient la solution ayant une valeur plus basse
  • de::SelectionStrategy, la stratégie de sélection *
    • de::SelectionBestFitness, effectue une sélection par la meilleure fitness, la stratégie par défaut
  • de::CrossoverStrategy, la stratégie de croisement *
    • de::CrossoverBinomial, croisement binomial, la stratégie par défaut
    • de::CrossoverExponential, croisement exponentiel
  • de::MutationStrategy, la stratégie de mutation *
    • de::MutationBest1, effectue une mutation par la meilleure solution, la stratégie par défaut
    • de::MutationRand1, effectue une mutation aléatoire
  • de::StoppingCriteriaStrategy, une stratégie d'arrêt *
    • de::StoppingCriteriaMaxGeneration, arrête l'évolution après un nombre maximum de générations, la stratégie par défaut

* Note importante : Il est important de réaliser que toutes les classes nommée avec un suffix Strategy sont des classes abstraites.

Approche de développement

Généralités

La bibliothèque est développée en C++20 et est compilée avec le compilateur Microsoft Visual Studio 2019.

Pour réaliser un projet, seulement deux classes doivent être instanciées :

  • de::DifferentialEvolution, le moteur de l'évolution différentielle (la classe centrale)
  • une classe héritant de de::SolutionStrategy qui représentera le problème à résoudre

Optionnellement, la création d'un objet de configuration de::EngineParameters permet la définition des paramètres spécifiques au moteur de l'évolution différentielle. À travers cet objet, il est possible de définir tous les constituants génériques de l'algorithme d'évolution différentielle. Sinon, des paramètres par défaut sont utilisés.

Il est attendu que l'objet personnalisé issu de la classe de::SolutionStrategy permettent la définition des paramètres propre au problème à résoudre. Toutefois, l'objet doit être préinitialisé avec ses propres paramètres viables au problème adressé.

Démarche d'utilisation

La forme la plus simple de l'utilisation de delib se fait suivant ces étapes :

  1. Rédaction d'une classe personnalisée héritant de de::SolutionStrategy adressant le problème à résoudre.
    • il est important de comprendre que plusieurs méthodes :
      • doivent être substituées (override)
      • peuvent être substituées (override)
    • un famille de classes présentent des exemples simples :
      • de::PeakFunctionMinSolution
      • de::PeakFunctionMaxSolution
      • de::PeakFunctionMinMaxSolution
  2. Instanciation d'un objet de::DifferentialEvolution avec l'objet personnalisé.
  3. Instanciation d'un objet de la classe créée en (1).
  4. Optionnellement, instanciation d'un objet de::EngineParameters pour la configuration des paramètres génériques de l'algorithme d'évolution différentielle.
  5. Configurer l'engin d'évolution différentielle (2) en lui donnant les objets créés en (3) et, optionnellement, en (4).
    • DifferentialEvolution::setEngineParameters
    • DifferentialEvolution::setup
  6. Lancer l'algorithme d'évolution différentielle (2).
    • DiffentialEvolution::process.

Exemples

Premier exemple

Cet exemple présente l'usage le plus simple de la bibliothèque.

Le problème à résoudre est l'identification des extrema de la célèbre fonction Peak de Matlab.

C'est la classe de::PeakFunctionMinMaxSolution qui représente le problème à résoudre. Cette classe hérite de de::SolutionStrategy et est disponible dans delib à titre d'exemple.

#include <iostream>
#include <DifferentialEvolution.h>
#include <PeakFunctionMinMaxSolution.h>
int main()
{
// Identification des éléments utilisés du namespace 'de'
using de::DifferentialEvolution;
using de::EngineParameters;
using de::PeakFunctionMinMaxSolution;
// Crée un objet DifferentialEvolution.
// Le moteur de l'algorithme d'évolution différentielle.
DifferentialEvolution deEngine;
// Configure l'algorithme d'évolution différentielle avec :
// - les paramètres génériques déterminés préalablement
// - le problème à résoudre :
// - ici on utilise l'exemple disponible dans la bibliothèque
// - PeakFunctionMinMaxSolution
// - on remarque l'allocation dynamique
deEngine.setup(new PeakFunctionMinMaxSolution);
// Lance l'algorithme d'évolution différentielle.
deEngine.process();
// Affiche les résultats.
std::string results;
results = deEngine.bestSolutionInfo();
std::cout << results;
return 0;
}

Deuxième exemple

Cet exemple présente la création d'une classe permettant de résoudre un problème personnalisé.

Le problème à résoudre est celui de la boîte ouverte. On développe une classe héritant de de::SolutionStrategy implémentant toutes les substitutions polymorphiques (override) requises pour une évolution bien ciblées.

On peut noter que la fonction de::SolutionStrategy::engineParameters n'est pas substituée et, par conséquent, l'implémentation par défaut est utilisée. Ceci veux dire que les paramètres par défaut sont ceux qui initialisant la solution.

#include <iostream>
#include <format>
#include <DifferentialEvolution.h>
#include <EngineParameters.h>
#include <OptimizationMaximization.h>
#include <FitnessIdentity.h>
#include <MutationRand1.h>
// Déclarations - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
// Fonction utilitaire pour suivre l'évolution de l'algorithme.
void printEvolutionInformation(size_t actualGeneration);
// Classe définissant une stratégie de résolution.
// On adresse ici le problème de la boite ouverte.
class OpenBoxStrategy : public de::SolutionStrategy
{
public:
// Constructeur personnalisé permettant de définir la forme de la feuille initiale.
OpenBoxStrategy(double width = 1.0, double height = 1.0);
// Substitution de la méthode toString() pour afficher les informations de la solution.
std::string toString(de::Solution const& solution) const override;
// Substitution de la méthode clone() pour permettre la duplication de la stratégie.
DEFINE_OVERRIDE_CLONE_METHOD(OpenBoxStrategy)
protected:
// Substitution de la méthode process() pour déterminer la valeur de la fonction objectif.
double process(de::Solution const& solution) override;
private:
double mWidth;
double mHeight;
};
int main()
{
// Crée du moteur de résolution.
de::DifferentialEvolution deEngine;
// Création et configuration de l'objet de paramétrisation.
de::EngineParameters deParameters;
deParameters.setPopulationSize(10); // Nombre d'individus dans la population.
deParameters.setMutationFactor(0.2); // Facteur de mutation.
deParameters.setCrossoverRate(0.8); // Taux de croisement.
deParameters.setMaxGeneration(150); // Nombre maximum de générations.
deParameters.setMutationStrategy(new de::MutationRand1); // Stratégie de mutation.
// On remarque que la détermination d'une stratégie passe par une allocation de mémoire dynamique.
// Configure l'algorithme d'évolution différentielle avec :
deEngine.setup(new OpenBoxStrategy);
deEngine.setEngineParameters(deParameters);
// Lance l'algorithme d'évolution différentielle.
// Utilise une fonction externe pour suivre l'évolution de l'algorithme.
deEngine.process(printEvolutionInformation);
// Affiche les résultats.
std::string results;
results = deEngine.bestSolutionInfo();
std::cout << results;
return 0;
}
// Définitions - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
void printEvolutionInformation(size_t actualGeneration)
{
std::cout << "Génération courate : " << actualGeneration << std::endl;
}
OpenBoxStrategy::OpenBoxStrategy(double width, double height)
: SolutionStrategy("un titre", "un sommaire", "une description")
// des chaînes de caractères plus pertinentes devraient être utilisées
, mWidth{ width }
, mHeight{ height }
{
// domain definition
mSolutionDomain.resize(1);
mSolutionDomain[0].set(0.0, std::min(height, width) / 2.0);
// strategies definition
setOptimizationStrategy(new de::OptimizationMaximization);
setFitnessStrategy(new de::FitnessIdentity);
}
std::string OpenBoxStrategy::toString(de::Solution const& solution) const {
// return une version texte de la solution
return std::format(R"...(
Taille de la feuille : {} x {}
Taille de la coupe : {:0.6f} x {:0.6f}
Volume de la boite : {:0.6f})...", mWidth, mHeight, solution[0], solution[0], solution.fitness());
}
double OpenBoxStrategy::process(de::Solution const& solution) {
// représente la coupe à réaliser
double cut{ solution[0] };
// retourne le volume de la boite obtenue
return cut * (mWidth - 2.0 * cut) * (mHeight - 2.0 * cut);
}
